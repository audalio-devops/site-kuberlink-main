"use client";

import { useEffect } from "react";
import { useRouter } from "next/navigation";

export default function AuthGuard({ children }: { children: React.ReactNode }) {
  const router = useRouter();

  useEffect(() => {
    // Função simples para ler cookies no cliente
    const hasToken = document.cookie
      .split("; ")
      .find((row) => row.startsWith("auth-token="));

    if (!hasToken) {
      // Se não houver token, redireciona para a Home
      router.push("/");
    }
  }, [router]);

  // Opcional: Adicionar um loading enquanto verifica
  return <>{children}</>;
}

Middleware: Protege a rota antes mesmo da página começar a carregar (lado do servidor).
AuthGuard (Client-side): A página começa a carregar no navegador e o JavaScript verifica o cookie para decidir se redireciona (lado do cliente). É a única forma de fazer isso em sites 100% estáticos.

Como saber se funcionou?

Por que usar o UID como ID do documento?
Ao usar o UID (loggedUser.uid) como nome do documento, você garante que cada usuário tenha exatamente um registro no banco de dados, facilitando buscas futuras e regras de segurança (ex: um usuário só pode ler o documento cujo ID seja igual ao seu próprio UID).

Localização do Banco de Dados: Escolha uma região próxima de você ou dos seus usuários. Se você estiver no Brasil, a opção southamerica-east1 (São Paulo) é a mais recomendada para garantir menor latência.
Regras de Segurança:
O Firebase perguntará se deseja iniciar no Modo de Produção ou Modo de Teste.
Para facilitar o desenvolvimento inicial, você pode escolher o Modo de Teste. Isso permitirá que seu site grave os documentos sem restrições nos primeiros 30 dias.
Importante: Depois que o banco estiver criado, você deve ir na aba "Regras" (Rules) e colar aquelas regras de segurança que conversamos anteriormente para proteger a coleção admins.
Finalizar: Após concluir o assistente, o Firestore estará pronto.
Assim que você ativar o banco de dados, tente fazer o login novamente no seu site. O código que já incluímos no Header.tsx se encarregará de:

Verificar se o usuário existe no Authentication.
Criar a coleção admins (ou clients) automaticamente.
Criar o documento com o UID do usuário.
Você não precisa criar as coleções manualmente; elas surgirão no console assim que o primeiro login for realizado com sucesso.
